1. Objetivo

El presente Plan de Adquisiciones tiene como objetivo identificar y gestionar los recursos externos necesarios para el desarrollo del proyecto FadeBooker.

Este plan permite definir qué elementos deben ser adquiridos, en qué condiciones y cómo serán utilizados durante el ciclo de vida del proyecto.

2. Alcance

Este plan considera la adquisición de recursos tecnológicos, servicios y herramientas necesarias para el desarrollo, pruebas y despliegue del sistema.

Incluye tanto recursos gratuitos como aquellos que podrían implicar un costo mínimo para la ejecución del proyecto.

3. Estrategia de Adquisición

La estrategia del proyecto se basa en el uso de herramientas gratuitas o con licencias académicas, con el objetivo de reducir costos.

En caso de ser necesario, se considerarán servicios externos básicos que permitan asegurar el correcto funcionamiento del sistema, como dominio web o servicios de apoyo.

5. Criterios de Selección

Los recursos seleccionados se basan en los siguientes criterios:

Disponibilidad de versiones gratuitas o académicas

Facilidad de uso e integración

Compatibilidad con el entorno del proyecto

6. Gestión de Proveedores

Los servicios utilizados provienen de proveedores reconocidos en el área tecnológica, lo que garantiza estabilidad y soporte en el desarrollo del sistema.

En caso de requerir servicios adicionales, se evaluarán nuevas opciones según costo, disponibilidad y compatibilidad.

7. Control de Adquisiciones

El seguimiento de las adquisiciones será responsabilidad del equipo del proyecto, asegurando que:

Los recursos estén disponibles cuando se necesiten

Se utilicen de manera eficiente

No se generen costos innecesarios

8. Conclusión

El Plan de Adquisiciones permite organizar y controlar los recursos necesarios para el desarrollo del sistema FadeBooker.

Gracias al uso de tecnologías de código abierto, servicios cloud con créditos académicos y herramientas gratuitas, se logra minimizar el costo del proyecto, manteniendo al mismo tiempo la calidad, escalabilidad y funcionalidad del sistema.
